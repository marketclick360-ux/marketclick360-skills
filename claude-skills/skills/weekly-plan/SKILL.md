# Weekly Plan

Not a fantasy productivity schedule — a peaceful, realistic plan the user can
actually live, with spiritual anchors protected first and explicit decisions
about what to cut, finish, and ignore.

## Steps

1. Read [reference/master-prompt.md](reference/master-prompt.md) **in full**
   and adopt it as your operating spec. It contains the non-negotiable
   spiritual anchors, weekly rhythm, folder system, project areas, operating
   rules, and the exact 12-section output format.
2. Ask **only** the three intake questions from the spec (unusual
   obligations this week; top 1–3 outcomes; this week's emphasis). If the
   user passed a capacity color as `$ARGUMENTS`, take it as given.
3. Produce the plan in the spec's 12-section format. Non-negotiables to
   enforce even if the user's stated outcomes conflict:
   - Nothing scheduled over a spiritual anchor, ever.
   - Thursday morning is spiritual prep only; Friday afternoon/evening and
     Sunday after meeting stay work-free unless explicitly permitted.
   - Every plan includes a **Finishing List** (move things to End Products),
     a **Not This Week** list, and a YouTube research boundary — the
     decision-pressure sections are not optional.
   - If the week is overloaded, say clearly what must be cut. Kind, direct.

## Decision pressure (why this skill exists)

A schedule alone is not the deliverable. Each week must move work through the
funnel: thinking (`01 Atlas`) → building (`02 Projects`) → finished
(`03 End Products`), with distractions parked in `04 Archive`. If nothing is
finishing and nothing is being declined, the plan has failed — tighten it.

## Works with the other skills

- **brain-dump** feeds this: if there's a fresh dump, process it first; its
  organized tasks are the raw material for the Top 3 Outcomes and schedule.
- **three-folder-system** executes section 6: the folder assignments made
  here become actual file moves via that skill's triage flow.
- The **Sunday Reset Review** (section 12) doubles as brain-dump's weekly
  review — run them together on Sunday afternoon.

## Output

Done means: all 12 sections delivered, anchors visibly protected, at least
one thing headed to End Products, and a Not This Week list the user agreed
to. Then stop — the user transfers the schedule to their own planner.
