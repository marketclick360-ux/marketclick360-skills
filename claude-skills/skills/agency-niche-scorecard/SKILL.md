# Niche Decision Scorecard

A standing quarterly review, not a one-shot. Deciding to defer IS a valid
outcome — re-run in 30 days.

## Steps

1. List every vertical actually served (clients + serious pipeline).
2. Score each 1–5 per criterion in [reference/scorecard.md](reference/scorecard.md),
   pulling from the listed data sources (time log, build-log reuse %, backlog,
   vault notes). Don't score from impressions when data exists.
3. Apply weights, rank, and test against the decision rules (also in the
   reference file) — minimum evidence thresholds, niche-down conditions, and
   the explicit stay-generalist conditions.
4. Output: the scored table, the recommendation (niche down on X / stay
   generalist, re-run {DATE}), and what evidence would change the answer.

## Guardrails

- One good client is an anecdote, not a niche. Minimum: 2 clients in a
  vertical OR 1 client + 3 same-vertical pipeline conversations.
- The operator makes the final call — this skill only arms it.
