# Niche Scorecard

Score each served vertical 1–5 per criterion, multiply by weight, sum.

| Criterion | Weight | Data source |
|---|---|---|
| Revenue per operator-hour (MRR ÷ operator hours on that client) | ×3 | Time log + billing |
| Workflow reusability (build-log reuse % for the vertical) | ×3 | Automation Builder build logs |
| Sales friction (discovery→signed days; discount pressure; payment behavior) | ×2 | Pipeline dates + vault notes |
| Existing asset leverage (repos, domain knowledge, audience fit) | ×2 | Manual, once |
| Expansion evidence (backlog depth, upgrade triggers fired) | ×2 | Repricing logs / client backlogs |
| Content resonance (inbound/engagement from that vertical's content) | ×1 | LinkedIn + inbound source notes |
| Regulatory drag (compliance constraints on agents) | ×(−2) | Intake Section E |

## Decision rules

**Niche down when ALL of:**
- One vertical leads by ≥25% on weighted score
- Its reuse % is rising across installs
- Its content pulls inbound

**Stay generalist longer when ANY of:**
- Scores cluster within ~15%
- The leader is a single client
- Reuse % is climbing across ALL verticals (the library is generalizing —
  fine at this stage)

**Minimum evidence to decide at all:** ≥2 clients in a vertical OR 1 client +
3 same-vertical pipeline conversations.

## After the decision

Niche chosen → content narrows to the vertical, the skills library deepens
vertically, installs drop toward 2–3 operator-hours because workflow N+1 is a
copy of workflow N. Deferred → re-run in 30 days with the same table.
