# Content Engine

Delivery IS the content pipeline: build logs, shadow-week reports, and weekly
reports are the raw material. Never start from a blank page — and never
publish without the operator's edit.

## Steps

1. Mine this week's delivery artifacts for 2–3 angles; the operator picks one
   and supplies a 5–10 minute raw take (voice note or Loom).
2. Draft one pillar piece + 3–4 LinkedIn posts in the operator's voice, using a
   format from [reference/formats-and-rules.md](reference/formats-and-rules.md).
3. Run the anonymization pass against the rules in the same file. Flag anything
   uncertain rather than deciding it's fine.
4. Queue everything for operator edit. Every piece ends with the standing CTA:
   "I take on 1 new business per month — intake link."

## Hard rules

- Anonymized lessons need no client sign-off; **named** case studies always
  need a signed release.
- Every piece must contain one specific, earned detail. No generic AI-hype.
- Outcomes over features: "recovered ~$9k/mo in missed follow-ups," not
  "uses MCP and sandboxed VMs."

## Client vault

Client context (facts, tone, tools, decisions, backlog) lives in the private
`marketclick360-ux/agency-vault` repo, operated per the `agency-client-vault`
skill. Read the client's `_CLIENT.md` card first — and `tone.md` before
drafting anything client-facing. Write decisions, commitments, and requests
back per that skill's Two-Output and propagation rules. Client data never
enters this public skills repo.
